🔐Authentication Flow

Step-by-step login journey from user to dashboard

1

User Submits

Email & Password entered on Login page

2

Supabase Auth

signInWithPassword() validates credentials

3

JWT Issued

JWT access token returned & stored in AuthContext

4

API Request

Frontend sends Bearer JWT in HTTP header

5

Middleware

verifySupabaseToken.js decodes & validates token

6

Role Lookup

MongoDB user fetched → role attached to req.user

7

Dashboard

Correct dashboard rendered based on role

📅Booking Workflow (Guest Journey)

Complete flow from room browsing to confirmation

1

Guest Visits Site Public

Lands on Home page → views featured rooms and hotel info

2

Login / Register Supabase Auth

New user registers → existing user logs in → JWT token stored in session

3

Browse Rooms

GET /api/rooms → rooms list with filters (type, price, availability)

4

Select Room & Dates

View Room Details → choose check-in / check-out → confirm booking

5

Backend Validates POST /api/bookings

Checks room availability → creates Booking document in MongoDB

6

Payment POST /api/payments

Payment record created → room status updated to "booked"

7

✅ Booking Confirmed!

Notification sent → Guest dashboard updated → Audit log recorded

🗄Database Models (MongoDB)

7 Mongoose collections and their key fields

👤 User

  • supabaseIdPK
  • nameString
  • emailString
  • roleadmin|staff|guest
  • phoneString

🏠 Room

  • _idPK
  • nameString
  • typeString
  • priceNumber
  • statusavailable|booked
  • amenities[String]

📅 Booking

  • _idPK
  • userIdFK→User
  • roomIdFK→Room
  • checkInDate
  • checkOutDate
  • statusString

💳 Payment

  • _idPK
  • bookingIdFK→Booking
  • amountNumber
  • statusString
  • dateDate

🔧 ServiceRequest

  • guestIdFK→User
  • staffIdFK→User
  • typeString
  • statusString
  • descriptionString

🔔 Notification

  • userIdFK→User
  • messageString
  • isReadBoolean
  • createdAtDate

📋 AuditLog

  • userIdFK→User
  • actionString
  • entityString
  • timestampDate
